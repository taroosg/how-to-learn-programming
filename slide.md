## 時間をどうやって作るか

--

（時間は）ないです．

--

人間の 24 時間は埋まっている．

--

時間は「引き算」で考える．

--

まずは 1 日のスケジュールを可視化する．

--

例 1

--

```
 6h            8h             2h    3h      5h
 睡眠           仕事           通勤   食事    その他
------------|----------------|----|------|----------
```

--

絶対に必要な時間とそれ以外を分ける．

--

「それ以外」を全部プログラミングにつぎ込む

--

```
 6h            8h             2h    3h      5h
 睡眠           仕事           通勤   食事    コード
------------|----------------|----|------|----------
```

--

「やらないこと」を明確にせよ！

--

プログラミングは「最優先事項」！

--

例 2

--

```
 6h             11h                  3h     3h     1h
 睡眠            仕事                 通勤    食事   その他
------------|----------------------|------|------|--
```

--

時短する

--

```
 6h             9h              3h     3h     3h
 睡眠            仕事             通勤   食事    コード
------------|------------------|------|------|------
```

--

これが「質」の議論

--

同じ「量」をより少ない時間で実現すること

--

1. 量（経験値）を準備する．

2. 「経験値」を用いて時間を「圧縮」する．

--

「量」があれば「質」を議論できる

--

だから「量 > 質」

--

優先順位（やらないこと）を決める．

質を高めることで時間を作り出せる．
